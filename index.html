Mulai dari nol

Install tools, kenalan Route-Controller-Model-View, sampai form laporan barang lengkap dengan upload foto.

00Apa itu Laravel, dan apa yang mau kita bikin

Laravel itu framework PHP. Anggap aja PHP polos itu bahan mentah, Laravel itu dapur yang udah lengkap: kompor, panci, pisau udah disiapin. Lo tinggal masak.

Yang dibikin di Level 1: website Lost and Found kampus. Dua arah:

  • Orang yang kehilangan barang bikin laporan: barangnya apa, hilang di mana, kapan, ada fotonya nggak
  • Orang yang menemukan barang juga bikin laporan: barang apa yang dia temuin, ditemukan di mana, sekarang dititipin di mana

Dua-duanya masuk ke satu daftar yang bisa dicari semua orang.

plaintext
User login -> pilih Kehilangan atau Menemukan -> isi detail + upload foto
                                                        |
                                        data masuk database -> muncul di daftar publik

Level 1 belum ada admin, belum ada klaim, belum ada verifikasi. Itu Level 2.

Empat istilah yang harus lo hafal duluan:

Istilah Tugasnya Analogi
Route nentuin URL apa manggil apa alamat rumah
Controller otak, ngatur logika resepsionis
Model wakil satu tabel database pegawai arsip
View tampilan yang dilihat user ruang tamu

Alurnya selalu Route -> Controller -> Model -> View. Sekali lo paham empat kotak ini, sisanya cuma variasi.

Install berurutan, jangan diloncat. Semua gratis.

Urutan install: Laragon dulu, terus Composer, Node, Git, terakhir VS Code.

Waktu install Git for Windows, semua pilihan Next aja, kecuali satu: pas ditanya default editor, pilih Visual Studio Code. Git Bash otomatis ikut keinstall, itu terminal yang bakal kita pakai terus.

Selesai install, buka Git Bash (klik kanan di desktop, pilih Open Git Bash Here) terus ketik satu-satu:

bash
php -v
composer -V
node -v
npm -v
git --version

Kelima-limanya harus keluar nomor versi. Kalau ada yang command not found, tutup Git Bash, buka lagi. Masih error? Berarti tool itu belum masuk PATH, install ulang dan centang opsi "Add to PATH".

PHP harus versi 8.3 ke atas. Kalau Laragon lo bawa PHP lama, download yang baru dari windows.php.net/download terus taro di folder laragon/bin/php/.

Karena project ini banyak main foto, cek juga extension GD-nya nyala:

bash
php -m | grep -i gd

Kalau gd nggak keluar, buka php.ini di folder PHP lo, cari baris ;extension=gd, hapus titik komanya, simpan, restart Laragon.

05Database dan migration

Buka Laragon, klik Start All. Klik tombol Database, bikin database baru namanya barang_hilang.

Edit file .env:

ini
DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=barang_hilang
DB_USERNAME=root
DB_PASSWORD=

Tes: php artisan migrate. Muncul daftar tabel yang dibikin? Nyambung.

Migration itu cetak biru tabel dalam bentuk kode. Kenapa nggak bikin manual di phpMyAdmin? Karena kalau kodenya, temen lo tinggal php artisan migrate dan dapet struktur yang persis sama.

bash
php artisan make:model ItemReport -m

-m artinya sekalian bikin migration. Buka file baru di database/migrations/, isi up():

php
public function up(): void
{
    Schema::create('item_reports', function (Blueprint $table) {
        $table->id();
        $table->foreignId('user_id')->constrained()->cascadeOnDelete();
        $table->string('report_type');
        $table->string('category');
        $table->string('item_name');
        $table->text('description');
        $table->string('color', 50)->nullable();
        $table->string('brand', 100)->nullable();
        $table->string('location');
        $table->date('event_date');
        $table->string('storage_place')->nullable();
        $table->string('contact');
        $table->string('photo_path')->nullable();
        $table->string('status')->default('open');
        $table->timestamps();
    });
}

Jalanin php artisan migrate lagi. Cek di phpMyAdmin, tabelnya ada.

Arti kolom yang penting:

  • report_type isinya hilang atau ditemukan. Satu tabel dipakai buat dua jenis laporan, biar nggak perlu bikin dua tabel yang isinya mirip.
  • location artinya beda tergantung jenis laporan: kalau hilang, ini tempat terakhir kali barangnya kelihatan. Kalau ditemukan, ini tempat barangnya ketemu.
  • storage_place cuma dipakai laporan ditemukan: sekarang barangnya dititipin di mana, misal "Pos Satpam Gedung B". Makanya nullable().
  • photo_path cuma nyimpen alamat filenya, contoh items/abc123.jpg. Fotonya sendiri disimpen di folder, bukan di database.
  • status default open, nanti bisa berubah jadi returned kalau barangnya udah balik ke pemiliknya.

Nama kolom database pakai snake_case (huruf kecil, pisah underscore). Ini wajib, karena Laravel nyarinya format itu.

06Model

Model itu wakil satu tabel. Satu tabel, satu model. Nama tabel jamak (item_reports), nama model tunggal (ItemReport).

app/Models/ItemReport.php:

php
<?php

namespace App\Models;

use Illuminate\Database\Eloquent\Model;
use Illuminate\Database\Eloquent\Relations\BelongsTo;

class ItemReport extends Model
{
    protected $fillable = [
        'user_id', 'report_type', 'category', 'item_name', 'description',
        'color', 'brand', 'location', 'event_date', 'storage_place',
        'contact', 'photo_path', 'status',
    ];

    protected function casts(): array
    {
        return ['event_date' => 'date'];
    }

    public function user(): BelongsTo
    {
        return $this->belongsTo(User::class);
    }

    public function photoUrl(): string
    {
        return $this->photo_path
            ? asset('storage/' . $this->photo_path)
            : asset('images/no-photo.png');
    }
}

$fillable itu daftar kolom yang boleh diisi dari form. Kalau lupa masukin satu kolom ke sini, datanya bakal diem-diem nggak kesimpen. Ini error paling sering buat pemula.

photoUrl() itu method bantuan. Daripada nulis asset('storage/' . ...) berulang-ulang di banyak view, cukup panggil $report->photoUrl(). Kalau fotonya kosong, otomatis nampilin gambar placeholder.

Tambahin relasi kebalikannya di app/Models/User.php, di dalam class:

php
public function itemReports(): \Illuminate\Database\Eloquent\Relations\HasMany
{
    return $this->hasMany(ItemReport::class);
}

Sekarang $user->itemReports ngasih semua laporan punya user itu. Nggak perlu nulis SQL sama sekali.

08Siapin tempat penyimpanan foto

Ini bagian yang beda dari project biasa, jadi baca pelan-pelan.

Foto yang diupload user disimpen di storage/app/public/. Masalahnya, folder storage itu nggak bisa dibuka langsung dari browser demi keamanan. Solusinya, Laravel bikin jalan pintas:

bash
php artisan storage:link

Perintah ini bikin pintu tembus dari public/storage ke storage/app/public. Jalanin sekali aja per project.

Di Windows, kalau muncul error soal symlink, tutup Git Bash, klik kanan ikonnya, pilih Run as administrator, terus jalanin ulang perintahnya. Alternatif lain, nyalain Developer Mode di Settings Windows.

Cek berhasil: harusnya muncul folder baru public/storage.

Terakhir, siapin gambar cadangan buat laporan yang nggak ada fotonya. Taro file gambar apa aja, kasih nama no-photo.png, di folder public/images/. Bikin foldernya kalau belum ada.

Tiga tempat yang gampang ketuker, hafalin bedanya:

Folder Isinya Boleh dibuka browser
public/images/ gambar bawaan aplikasi, misal logo ya
storage/app/public/ file yang diupload user lewat pintu tembus tadi
storage/app/private/ file rahasia, misal scan KTP tidak

09Controller dan upload foto

bash
php artisan make:controller ItemReportController

Isi app/Http/Controllers/ItemReportController.php lengkap:

php
<?php

namespace App\Http\Controllers;

use App\Models\ItemReport;
use Illuminate\Http\RedirectResponse;
use Illuminate\Http\Request;
use Illuminate\Support\Facades\Storage;
use Illuminate\View\View;

class ItemReportController extends Controller
{
    public function index(Request $request): View
    {
        $reports = ItemReport::with('user')
            ->when($request->type, fn ($query, $type) => $query->where('report_type', $type))
            ->when($request->q, fn ($query, $q) => $query->where('item_name', 'like', "%{$q}%"))
            ->latest()
            ->paginate(12)
            ->withQueryString();

        return view('pages.laporan.index', compact('reports'));
    }

    public function create(): View
    {
        return view('pages.laporan.create', [
            'categories' => config('items.categories'),
            'types' => config('items.types'),
        ]);
    }

    public function store(Request $request): RedirectResponse
    {
        $data = $request->validate([
            'report_type' => ['required', 'in:hilang,ditemukan'],
            'category' => ['required', 'in:' . implode(',', array_keys(config('items.categories')))],
            'item_name' => ['required', 'string', 'max:120'],
            'description' => ['required', 'string', 'min:15', 'max:1000'],
            'color' => ['nullable', 'string', 'max:50'],
            'brand' => ['nullable', 'string', 'max:100'],
            'location' => ['required', 'string', 'max:150'],
            'event_date' => ['required', 'date', 'before_or_equal:today'],
            'storage_place' => ['nullable', 'required_if:report_type,ditemukan', 'string', 'max:150'],
            'contact' => ['required', 'string', 'max:100'],
            'photo' => ['nullable', 'image', 'mimes:jpg,jpeg,png,webp', 'max:2048'],
        ]);

        if ($request->hasFile('photo')) {
            $data['photo_path'] = $request->file('photo')->store('items', 'public');
        }

        $request->user()->itemReports()->create($data);

        return to_route('laporan.index')->with('success', 'Laporan berhasil dikirim.');
    }

    public function show(ItemReport $laporan): View
    {
        return view('pages.laporan.show', ['report' => $laporan->load('user')]);
    }

    public function destroy(ItemReport $laporan): RedirectResponse
    {
        abort_unless($laporan->user_id === request()->user()->id, 403);

        if ($laporan->photo_path) {
            Storage::disk('public')->delete($laporan->photo_path);
        }

        $laporan->delete();

        return to_route('laporan.index')->with('success', 'Laporan dihapus.');
    }
}

Yang penting dari kode di atas:

  • 'photo' => ['nullable','image','mimes:jpg,jpeg,png,webp','max:2048']. image mastiin yang diupload beneran gambar, bukan file .exe yang cuma diganti namanya. max:2048 itu 2048 kilobyte, alias 2 MB. Satuannya KB, bukan MB, sering ketuker.
  • store('items', 'public') nyimpen file ke storage/app/public/items/ dengan nama acak, terus balikin alamatnya. Nama acak itu penting: kalau dua orang upload foto.jpg, nggak saling nimpa.
  • Kolomnya di form namanya photo, tapi di database photo_path. Yang disimpen ke database cuma teks alamatnya.
  • required_if:report_type,ditemukan bikin kolom tempat penitipan wajib diisi cuma kalau laporannya jenis "menemukan".
  • Di destroy(), fotonya ikut dihapus. Kalau nggak, filenya jadi sampah yang numpuk selamanya di server.

Daftarin route di routes/web.php:

php
use App\Http\Controllers\ItemReportController;

Route::get('/', [ItemReportController::class, 'index'])->name('laporan.index');
Route::get('/laporan/{laporan}', [ItemReportController::class, 'show'])->name('laporan.show');

Route::middleware('auth')->group(function () {
    Route::get('/laporan-baru', [ItemReportController::class, 'create'])->name('laporan.create');
    Route::post('/laporan', [ItemReportController::class, 'store'])->name('laporan.store');
    Route::delete('/laporan/{laporan}', [ItemReportController::class, 'destroy'])->name('laporan.destroy');
});

Perhatiin: daftar dan detail laporan ditaro di luar middleware('auth'), jadi bisa dilihat siapa aja tanpa login. Yang butuh login cuma bikin dan hapus laporan. Ini disengaja, soalnya orang yang kehilangan barang belum tentu punya akun.

Cek semua route kebaca:

bash
php artisan route:list

Tiga hal yang wajib dipahami:

  • enctype="multipart/form-data" di tag <form>. Wajib kalau ada upload file. Kalau lupa, filenya nggak ikut kekirim dan lo bakal bingung kenapa $request->hasFile('photo') selalu false padahal udah milih gambar.
  • Input file nggak pakai komponen Breeze, soalnya Breeze nggak punya. Pakai <input type="file"> biasa, dandanin pakai class file: bawaan Tailwind.
  • @csrf itu token keamanan. Tiap form POST harus punya ini, kalau nggak Laravel nolak dengan error 419.

old('nama_field') nampilin lagi isian sebelumnya kalau validasi gagal. Sayangnya ini nggak berlaku buat input file, jadi kalau validasi gagal user harus milih fotonya lagi. Itu batasan browser, bukan bug.

07Klaim barang

Ini inti sistemnya. Orang yang ngerasa barangnya ada di laporan temuan, ngajuin klaim plus bukti.

bash

php
public function up(): void
{
    Schema::create('claims', function (Blueprint $table) {
        $table->id();
        $table->foreignId('item_report_id')->constrained()->cascadeOnDelete();
        $table->foreignId('user_id')->constrained()->cascadeOnDelete();
        $table->text('proof_description');

proof_description itu bukti kepemilikan berupa tulisan, misal isi dompetnya apa aja. proof_path buat upload bukti tambahan, misal foto pas lagi make barang itu.

Controller-nya, dikasih store() doang:

bash
php artisan make:controller ClaimController
php
public function store(Request $request, ItemReport $report): RedirectResponse
{
    abort_unless($report->report_type === 'ditemukan' && $report->status === 'open', 422);
    abort_if($report->user_id === $request->user()->id, 403, 'Nggak bisa klaim laporan sendiri.');

    $data = $request->validate([
        'proof_description' => ['required', 'string', 'min:20', 'max:1000'],
        'proof' => ['nullable', 'image', 'mimes:jpg,jpeg,png,webp', 'max:2048'],
    ]);

    if ($request->hasFile('proof')) {
        $data['proof_path'] = $request->file('proof')->store('proofs', 'public');
    }

    $report->claims()->create($data + ['user_id' => $request->user()->id]);

    return back()->with('success', 'Klaim dikirim. Tunggu diverifikasi petugas.');
}

Dua baris abort di atas itu yang bikin sistemnya nggak gampang disalahgunakan: cuma laporan temuan yang masih terbuka yang bisa diklaim, dan orang nggak bisa ngeklaim laporannya sendiri.

LATIHAN

- bikin sendiri:

  • Relasi claims() di model ItemReport
  • Route POST buat klaim, taro di dalam middleware('auth')
  • Form klaim di halaman detail, muncul cuma kalau laporannya jenis temuan dan statusnya masih open

08Halaman petugas

bash
php artisan make:controller Petugas/ClaimReviewController

Dikasih index() dan approve() doang:

php
<?php

namespace App\Http\Controllers\Petugas;

use App\Http\Controllers\Controller;
use App\Models\Claim;
use Illuminate\Http\RedirectResponse;
use Illuminate\Http\Request;
use Illuminate\View\View;

class ClaimReviewController extends Controller
{
    public function index(Request $request): View
    {
        $claims = Claim::with('user', 'itemReport.category')
            ->when($request->status, fn ($query, $status) => $query->where('status', $status))
            ->latest()
            ->paginate(15);

        return view('pages.petugas.klaim.index', compact('claims'));
    }

    public function approve(Claim $claim): RedirectResponse
    {
        abort_unless($claim->status === 'pending', 422);

        $claim->update(['status' => 'approved']);
        $claim->itemReport->update(['status' => 'returned']);

        $claim->itemReport->claims()
            ->where('id', '!=', $claim->id)
            ->where('status', 'pending')
            ->update(['status' => 'rejected', 'officer_note' => 'Barang sudah diklaim pihak lain.']);

        return back()->with('success', 'Klaim disetujui, barang ditandai sudah dikembalikan.');
    }
}

Method approve() ngerjain tiga hal sekaligus: setujui klaimnya, tandai barangnya udah balik, dan tolak otomatis klaim lain buat barang yang sama. Tanpa langkah ketiga, satu barang bisa diklaim dua orang dan petugas bingung sendiri.

->with('user', 'itemReport.category') itu eager loading. Titik di itemReport.category artinya ambil relasi bertingkat sekalian. Tanpa ini, nampilin 15 baris butuh puluhan query database, bukan tiga. Namanya masalah N+1, dan ini penyebab web lemot nomor satu.

Route-nya:

php
use App\Http\Controllers\Petugas\ClaimReviewController;

Route::middleware(['auth', 'petugas'])->prefix('petugas')->name('petugas.')->group(function () {
    Route::get('klaim', [ClaimReviewController::class, 'index'])->name('klaim.index');
    Route::patch('klaim/{claim}/approve', [ClaimReviewController::class, 'approve'])->name('klaim.approve');
});
LATIHAN

- bikin sendiri:

  • Method reject(): wajib validasi officer_note minimal 5 karakter, status klaim jadi rejected, dan status laporannya jangan diubah
  • View pages/petugas/klaim/index.blade.php: tabel berisi nama pengklaim, nama barang, bukti tulisan, thumbnail bukti foto, plus tombol Setujui dan Tolak
  • Menu "Verifikasi Klaim" di navbar, dibungkus @if (auth()->user()->isPetugas()) biar pelapor biasa nggak lihat

Contoh bentuk tombol setujui, buat contekan form method PATCH:

    @csrf
    @method('PATCH')

Git & GitHub

Simpen kodenya online

Pakai Git Bash buat nyimpen riwayat kode, push ke GitHub, dan pastiin foto user nggak ikut kebawa.

01Kenapa perlu Git

Git itu tombol undo raksasa buat kode. Tiap perubahan disimpen, dan lo bisa balik ke versi kapan pun.

GitHub itu tempat nyimpen Git lo di internet. Gunanya: kode nggak hilang kalau laptop rusak, bisa dikerjain bareng, dan jadi portofolio yang bisa dilihat orang.

Beda dua-duanya: Git jalan di laptop lo, GitHub di internet. Bisa pakai Git tanpa GitHub, tapi jangan.

02Setelan awal

Buka Git Bash, jalanin sekali seumur hidup:

bash
git config --global user.name "Nama Lo"
git config --global user.email "email@yang-dipakai-daftar-github.com"
git config --global init.defaultBranch main

Email-nya harus sama dengan yang lo pakai daftar GitHub, biar commit lo keitung.

Cek udah bener: git config --list.

03Jangan sampai kepush

Dua hal yang haram masuk GitHub di project ini.

Pertama, file .env. Isinya password database dan kunci aplikasi.

Kedua, foto yang diupload user. Ini penting khusus buat project kayak gini. Isi folder storage/app/public/ itu data, bukan kode. Kalau ikut kepush, repo lo bakal bengkak ratusan megabyte, dan foto barang orang lain jadi bisa diakses siapa aja yang buka GitHub lo.

Laravel udah otomatis bikin .gitignore yang ngeblok .env, vendor/, node_modules/, dan isi storage/. Cek dulu:

bash
cat .gitignore

Pastiin ada baris .env dan /storage/*.key. Buat aman, tambahin sendiri baris ini di paling bawah:

plaintext
/storage/app/public/*
!/storage/app/public/.gitkeep
/public/storage

Baris pertama ngeblok semua isi folder upload. Baris kedua tetep ngizinin satu file penanda biar foldernya nggak hilang dari repo. Baris ketiga ngeblok pintu tembus hasil storage:link, karena itu harus dibikin ulang di tiap komputer.

vendor/ dan node_modules/ juga diblokir karena isinya ribuan file hasil download. Orang lain tinggal composer install dan npm install buat dapetin isinya lagi.

04Commit pertama

Masuk ke folder project dulu:

bash
cd /c/laragon/www/barang-hilang
git init
git status

git status nampilin file apa aja yang berubah. Warna merah artinya belum disimpen Git. Kalau di situ ada nama file foto .jpg, berarti .gitignore lo belum bener. Benerin dulu, jangan lanjut.

bash
git add .
git commit -m "Setup awal project Lost and Found"
  • git add . artinya "siapin semua perubahan"
  • git commit -m "..." artinya "simpen, dengan catatan ini"

Pesan commit ditulis jelas: Tambah upload foto laporan, bukan update atau asdf. Nanti lo sendiri yang bingung.

Lihat riwayatnya: git log --oneline.

05Bikin repo di GitHub dan push

Buka github.com/new. Isi:

  • Repository name: barang-hilang
  • Pilih Public atau Private, bebas
  • Jangan centang "Add a README file". Biarin kosong, soalnya project kita udah ada isinya.

Klik Create repository. GitHub bakal nampilin alamat repo-nya, bentuknya https://github.com/username/barang-hilang.git.

Balik ke Git Bash:

bash
git remote add origin https://github.com/username/barang-hilang.git
git push -u origin main

Ganti username sama username GitHub lo. Pas pertama kali push, muncul jendela login GitHub. Login lewat browser aja, paling gampang.

Refresh halaman GitHub-nya. Kode lo udah ada di situ.

Kalau muncul error src refspec main does not match any, berarti branch lo masih namanya master. Ganti dengan git branch -M main, terus push lagi.

06Alur harian

Tiap selesai ngerjain satu fitur, ulangi tiga langkah ini:

bash
git add .
git commit -m "Tambah galeri foto di halaman detail"
git push

Cuma itu. git push doang, tanpa embel-embel, karena -u origin main tadi udah nyimpen tujuannya.

Kalau ngoding di komputer lain, ambil versi terbaru dulu: git pull.

07Branch

Branch itu jalur terpisah. Gunanya biar kalau lo bikin fitur baru dan ternyata kacau, kode utama tetep aman.

bash
git checkout -b fitur-klaim

Kerja seperti biasa, add dan commit di branch itu. Kalau udah beres dan mau digabung ke jalur utama:

bash
git checkout main
git merge fitur-klaim
git push

Pindah-pindah branch: git checkout main atau git checkout fitur-klaim. Lihat lo lagi di branch mana: git branch.

08Perintah darurat

Situasi Perintah
Salah edit file, belum di-commit git checkout -- namafile
Udah git add, mau dibatalin git reset namafile
Pesan commit terakhir salah tulis git commit --amend -m "Pesan baru"
Mau lihat perubahan apa aja git diff
Mau download project orang git clone <url-repo>

Jangan pakai git reset --hard kalau belum paham. Itu ngehapus kerjaan lo tanpa bisa dibalikin.

Kalau lo kadung ngepush foto atau .env, ngehapusnya nggak cukup pakai git rm doang, soalnya filenya masih nyimpen di riwayat. Cara paling aman buat pemula: ganti semua password yang bocor, terus bikin repo baru dari nol.

09Habis clone dari GitHub

Kalau lo atau orang lain clone project ini dari GitHub, foldernya belum bisa langsung jalan:

bash
composer install
npm install
cp .env.example .env
php artisan key:generate
php artisan migrate --seed
php artisan storage:link
npm run dev

Perhatiin ada php artisan storage:link di situ. Ini wajib dan gampang kelupaan, soalnya pintu tembus foto tadi memang sengaja nggak ikut di-push. Lupa jalanin ini, semua foto di web bakal jadi ikon gambar rusak.
